📚 Formatos Suportados

  • Valgrind Memcheck — valgrind --tool=memcheck --leak-check=full ./tfs 2> valgrind.log
  • AddressSanitizer — Compile com -fsanitize=address e redirecione stderr

🎯 Classificação de Severidade

  • 🔴 CRITICAL — Invalid write / use-after-free write: corrompe heap, pode causar crash imprevisível
  • 🟡 HIGH — Invalid read: lê memória já liberada, comportamento indefinido
  • ⚠️ WARNING — Use of uninitialised value / cond jump: lógica incorreta
  • 💧 LEAK (definitely lost) — Memória alocada e nunca liberada
  • 🔵 LEAK (possibly/indirectly) — Provável não-problema em servidores long-running

🔧 Comando Recomendado (Valgrind)

valgrind \\
  --tool=memcheck \\
  --leak-check=full \\
  --show-leak-kinds=all \\
  --track-origins=yes \\
  --error-exitcode=1 \\
  ./tfs 2> valgrind-definitive.log

🔧 Comando Recomendado (ASan)

cmake .. -DCMAKE_CXX_FLAGS="-fsanitize=address -g" \\
         -DCMAKE_EXE_LINKER_FLAGS="-fsanitize=address"
ASAN_OPTIONS=log_path=asan.log ./tfs

💡 O que procurar primeiro

  • ❌ Invalid write em bloco free'd → crash imprevisível + corrupção de dados
  • ❌ Invalid read via internalMoveItem → crash ao mover items stackable
  • ❌ use-after-free via ScriptEnvironment → crash em TalkActions com item:clone()
  • ⚠️ Definitely lost bytes crescendo → memory leak real
