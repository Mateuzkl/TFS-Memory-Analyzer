❓ Como Usar

Guia de referência rápida para os formatos suportados.

📚 Formatos Suportados

  • Valgrind Memcheck — Gera um .log com erros de memória detalhados e stack traces.
  • AddressSanitizer (ASan) — Compile com -fsanitize=address e redirecione o stderr para um arquivo.

🔧 Comando Recomendado — Valgrind

valgrind \
  --tool=memcheck \
  --leak-check=full \
  --show-leak-kinds=all \
  --track-origins=yes \
  --error-exitcode=1 \
  ./server 2> valgrind.log

🔧 Comando Recomendado — ASan

cmake .. -DCMAKE_CXX_FLAGS="-fsanitize=address -g" \
         -DCMAKE_EXE_LINKER_FLAGS="-fsanitize=address"
ASAN_OPTIONS=log_path=asan.log ./server

🎯 Classificação de Severidade

  • 🔴 CRITICAL — Invalid write / use-after-free write: corrompe heap, pode causar crash imprevisível.
  • 🟡 HIGH — Invalid read: lê memória já liberada, comportamento indefinido.
  • ⚠️ WARNING — Use of uninitialised value / cond jump: lógica incorreta.
  • 💧 LEAK (definitely lost) — Memória alocada e nunca liberada.
  • 🔵 LEAK (possibly / indirectly) — Provável não-problema em processos de longa execução.

💡 O que priorizar primeiro

  • ❌ Invalid write em bloco free'd → crash imprevisível + corrupção de dados.
  • ❌ use-after-free → crash em operações com objetos já destruídos.
  • ❌ heap-buffer-overflow → leitura/escrita fora dos limites alocados.
  • ⚠️ Definitely lost crescendo → memory leak real que precisa ser corrigido.
